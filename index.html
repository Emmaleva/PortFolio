La figure 1 est une capture d'écran de l'outil de développement IDEA , un outil permettant aux développeurs d'accélérer et de faciliter le développement d'applications,
nottament dans le language de programmation Java.
J’ai utilisé cet outil pour créer avec mon groupe d'SAE un jeu de plateau nommé Orbito++.
La contrainte technique pour réaliser le jeu était d’utiliser une bibliothèque de classes Java nommée boardifier,
dédiée à la création de jeux de plateau, et qui suit le paradigme de programmation MVC (Modèle, Vue, Contrôleur).
Cette bibliothèque utilisant elle même une autre bibliothèque pour créer des interfaces graphiques nommé JavaFX.
Le paradigme de programmation MVC concerne les interfaces graphiques et implique de séparer les composants graphiques (Vue) des données (Modèle) du traitement de ces données (Controlleur).
Il est conseillé de structurer le modèle, la vue ainsi que le controlleur sous forme de package , un système de Java permettant de structurer et grouper les classes.
Dans la partie gauche de la figure 1, il est possible de voir certaines classes avec une flèche gauche rouge et une flèche verte droite , ces classes sont en vert sur la figure , ces classes représentes des classes de test pour tester que les classes de package aient bien le résultat et comportement attendu.
La Figure 1 illustre aussi le polymorphisme , qui consiste en la réécriture de méthodes déjà existantes dans la classe parent dans la classe fille, en effet dans la figure 1, la méthode playTurn est redéfinie dans la classe OrbitoController.
Ce projet nécessitait aussi de créer une configuration de lancement pour pouvoir modifier les classes tout en pouvant lancer le programme rapidement (en violet sur la Figure 1)

La Figure 2 est une capture d'écran de l'outil de développement PyCharm,
il s'agit d'un outil permettant de faciliter le développement comme IDEA, mais pour le language de programmation python.
J'ai utilisé cet outil , la bibliothèque de gestion de base de donnée pymysql ,ainsi que la bibliothèque flask pour réaliser un site internet de e-commerce.
La contrainte pour créer ce site était d'utiliser la biblothèque flask ainsi que la bibliothèque de base de données pymysql,
j'ai donc dû écrire des requếtes sql (en rouge sur la Figure 2) pour récupérer les bonnes informations, ainsi qu'éviter les injections SQL (rouge sur la figure 2)
J'ai aussi dû structurer mon projet correctement avec les "blueprint" pour gérer la logique de chaque page dans la partie "controllers" en bleu,
tout le contenu multimédia ainsi que les feuilles de style dans la partie "static" (en vert sur la figure 2) ainsi que toute la structure de la page en jaune sur la figure 2.
Pour pouvoir assurer le fonctionnement du site , il était aussi nécessaire de sélectionner le bon type de requête (GET pour récupérer des informations, Post pour en envoyer , en violet sur la Figure 2)
